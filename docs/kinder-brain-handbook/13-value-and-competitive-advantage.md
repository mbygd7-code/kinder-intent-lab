# 13. 가치와 경쟁력 — 과장 없이, 현재와 미래를 구분해서

> **한눈에 보기** — 킨더브레인의 경쟁력은 "LLM을 쓴다"가 아니라, **유아교육 도메인 온톨로지 + 검증 가능한 성능 체계 + 안전 실행 신호 + 교사 언어 데이터 자산**의 결합이다. 이 장은 각 항목을 "현재 근거(구현됨)"와 "미래 가능성(계획)"으로 나눠 적는다 — 투자·영업 자료로 옮길 때도 이 구분을 유지할 것.

---

## 1. 비교 우위 — 무엇과 다른가

| 대안 | 왜 부족한가 | 킨더브레인의 답 |
|---|---|---|
| 일반 LLM 직호출 | 답의 품질을 재는 자·개선 루프·안전 게이트가 없음. 프롬프트 수정=재배포뿐 | 동결 시험지 채점(88.1% 베이스라인 실측) + 승격 게이트 + 위험 신호 계약 |
| 룰베이스/키워드 | 생략·구어("오늘 찍은 거 부모님께 보여드리자")에 취약, 유지비 폭증 | 예문 검색+LLM 판정+성향 보정, 애매하면 clarify |
| 범용 intent router | 도메인 무지 — "관찰 포인트/목표 점검/기간 회고" 같은 유아교육 경계 구분 불가 | 70의도·혼동쌍 명시 온톨로지 + 경계 실측(confusion edge) |

## 2. 경쟁력 항목별 — 현재 근거 / 사용자 가치 / 모방 난이도 / 부족한 점

### ① 유아교육 특화 온톨로지 (현재 구현)
- 현재 근거: 8영역·70의도·의도별 정의/예문 419개/혼동쌍 70개 전 지정, major 개정 절차(STUDIO 신설 전례)까지 운영됨 [근거: seeds/ontology_v1.yaml, docs v1.5]
- 사용자 가치: 교사의 실제 업무 단위로 말이 연결됨(범용 분류가 아니라)
- 모방 난이도: 분류 체계 자체보다 **경계 정의·혼동쌍·검증 이력**이 자산 — 도메인 전문가+운영 반복 없이는 복제 어려움
- 부족: 62개 의도의 시험 문항 0 — 경계 주장의 실측 증거가 아직 CRITICAL 7에 편중

### ② 검증 가능한 성능 체계 (현재 구현)
- 현재 근거: 사람 저작·2인 검수·동결 시험지, 지표 동결 저장, zero-shot 베이스라인 88.1% 실측, 회귀 차단 승격 게이트 [근거: arena_runs, backend/app/arena/*, config::version_gate]
- 사용자 가치: "좋아졌다"를 느낌이 아니라 숫자로 — 도입 의사결정·감사에 강함
- 모방 난이도: 벤치마크를 **오염 없이**(TRAIN 격리, DB CHECK) 유지하는 규율은 후발주자가 흔히 실패하는 지점
- 부족: 아직 유효 점수 0회(대표 예문 대기) — "체계는 완비, 숫자는 곧"이 정직한 문장

### ③ 안전한 실행 (계약 구현 · UI 계획)
- 현재 근거: CRITICAL 7 자동 표시(requires_confirmation), CWAR 2%·CCC 80% 게이트 정의, 되묻기/abstain 정책 [근거: infer_response 계약, config::arena]
- 사용자 가치: 학부모 오발송·출결 오기재 같은 "한 번이 치명적인" 사고의 구조적 방지
- 모방 난이도: "정확도 경쟁"은 흔하지만 **CCC(되묻기 세탁 방지)까지 넣은 안전 지표 설계**는 드묾
- 부족: 실행 UI·실사용 오발 실측은 트랙 2

### ④ 설명 가능한 혼동 구조 (현재 구현)
- 현재 근거: 혼동 edge 629개(가설)·실측 확정 체계·3D 뇌 시각화·애매 발화 리포트(출처 분리) [근거: confusion_edges, frontend/src/brain3d, observatory::ambiguity_report]
- 사용자 가치: "왜 틀렸나"를 노드·연결로 보여줌 — 개선 우선순위가 화면에 보임
- 부족: confirmed edge 0 — 첫 유효 채점 후부터 채워짐

### ⑤ 지속 자기 강화 + 자가 채점 차단 (현재 구현)
- 현재 근거: 자동 증산·가설·측정·조건부 승격 + GOLD 인간 유일문 [근거: 08장]
- 가치: 데이터가 쌓일수록 좋아지되, 스스로 정답을 찍어 무너지는 붕괴 없음 — 장기 신뢰

### ⑥ 11월 첫날부터 작동하는 Seed Brain (핵심 전략 — 진행 중)
- 현재 근거: 파이프라인 완비, 콜드스타트 제거 전략 문서화 [근거: 02장]
- 가치: 경쟁 서비스가 "출시 후 학습"일 때, 우리는 출시일에 이미 기본 이해 제공
- 부족: Seed Brain의 유효 점수 확보가 남은 관문(현재 병목=검수)

### ⑦ 데이터 자산 — 기능 로그가 아니라 "교사의 사고와 표현" (현재 축적 시작)
- 현재 근거: 발화+화면 문맥+판정+사람 교정의 연결 저장(inference_request_id 감사 체인), provenance 4필드 [근거: gym/live.py, episodes CHECK]
- 가치: "어떤 버튼을 눌렀나"가 아니라 **"교사가 무엇을 어떻게 말하고 원하는가"**의 정량 자산 — 신기능 기획의 근거 언어가 됨(직원 의견서의 기대와 일치)
- 모방 난이도: 이 데이터는 서비스 운영 없이는 못 모음 — 시간이 해자
- 부족: 아직 내부 발화 중심(33건 GYM_HUMAN) — 실교사 분포는 트랙 2부터

### ⑧ 자연어 단일 인터페이스로의 통합 (미래 가치)
- 킨더버스 전 기능이 "한마디"로 연결되는 경험 — intent→slot→action 3층 중 intent층 완성, 나머지는 트랙 2 [상태: 1층 IMPLEMENTED / 2·3층 PLANNED — 12장]

## 현재 상태
- 과장 금지 요약: **"측정·안전·개선의 뼈대는 전부 실물이고, 점수와 실사용 증거는 지금 만드는 중"** — 이 문장이 대외적으로도 안전하다.

## 주의사항
- "96% 정확도"를 현재형으로 쓰지 말 것(목표). "0.0%"를 단독 인용하지 말 것(대표 예문 0의 정직한 0 + 베이스라인 88.1% 병기).
- 마케팅 화법 권장: 킨더브레인이라는 이름보다 **"말귀 알아듣는 킨더버스"라는 결과**로 설명(브레인은 기반 엔진).
