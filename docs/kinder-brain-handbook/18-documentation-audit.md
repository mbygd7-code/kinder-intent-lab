# 18. 문서 감사 — 이 핸드북 자체의 검증 기록

> **한눈에 보기** — 핸드북 18편(00~17)을 작성한 뒤 저장소와 재대조했다. 자동 검증기(링크·근거 경로·API path·config key·스크립트 실존·코드펜스)와 표본 수기 검수를 함께 돌렸고, 발견된 오류는 **문서에 먼저 반영**했으며(아래 §3), 남은 불확실성만 §4에 기록한다. 검증 시점: 2026-07-17.

---

## 1. 검증 방법

- **팩트팩 고정**: 집필 전 저장소 전수 인벤토리 + 로컬 DB 읽기 전용 스냅샷(SELECT만)을 단일 파일로 고정하고, 모든 수치는 그 파일만 인용(기억 인용 금지).
- **자동 재대조 스크립트**: 18편 전체에 대해 ① 핸드북 내부 링크 실존 ② `[근거: …]` 경로 실존 ③ 본문 `/v1/...` path를 실제 라우터(30개)와 대조 ④ 백틱 config key를 experiments.yaml(117 key)과 대조 ⑤ 스크립트명을 scripts/(20개)와 대조 ⑥ 코드펜스 짝. (스크립트는 세션 스크래치패드에 있으며 저장소를 수정하지 않음)
- **표본 수기 검수**: 색인(16장) 전문 정독, 각 문서 양식(한눈에 보기·근거·상태태그) 존재 확인, 구버전 수치(63의도·7영역) 전수 부재 확인, 핵심 주장 2건(GYM_CORRECTION 생성 경로·persona_warmup 소비처) grep 재검증.

## 4. 남은 불확실성 (UNKNOWN — 저장소만으로 확정 불가)

3. **미수집 테이블 행수 일부**(sources·situation_frames·persona 4표 등) — 05장 §5-6에 UNKNOWN 표기(주요 표는 전부 수집됨).
4. **저장소 밖 사실**: 킨더버스 측 준비 상태·Windows 미푸시 작업·배포 인프라(17장 G1~G3).
5. **런타임 스펙 원문 파일 부재** — README가 참조하는 kinder-intent-runtime-spec 본문이 저장소에 없음(17장 A2).
6. 본 감사는 **문서→코드 방향**의 전수였다. 코드→문서 방향(코드에 있는데 핸드북이 안 다룬 기능)의 누락 가능성은 16장 색인 범위(28행) 밖에서는 배제 못 함.
