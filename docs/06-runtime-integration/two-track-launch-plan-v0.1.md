## 1. 왜 투트랙인가 (콜드스타트 제거)

11월 킨더버스 오픈 후에 데이터를 모아 브레인을 만들면 초기 교사는 혜택이 없고, 첫
사용자들이 미완성 AI의 훈련자가 된다. → **트랙 1**(지금~11월): 실험실에서 선행 브레인
구축(전문가·직원·검수 데이터, KTIB·Arena로 증명). **트랙 2**(11월~): 선행 브레인을 탑재한
채 오픈하고, 실사용 데이터(PRODUCTION_SHADOW 채널)로 검수를 거쳐 지속 고도화.
연결 준비(§4)는 트랙 1과 병행한다 — 10월 말 시작은 늦다.

## 2. 의도별 적용 등급 A/B/C/HOLD

전역 스테이지(Shadow→Suggest→Assist)를 **의도 단위**로 세분한다. 등급은 사람이 정하는
값이 아니라 **측정에서 파생되는 읽기 전용 VIEW**다 — 원천은 Arena 산출(heldout_accuracy,
원칙 8과 동형)과 리스크 모델(rm-1.x)뿐이며, 훈련·교정 코드는 등급에 관여할 수 없다.

판정 규칙(우선순위 순 — 임계값은 구현 시 `config/experiments.yaml`에만 둔다, 원칙 7):

| 순서 | 조건 | 등급 |
|---|---|---|
| 1 | 리스크 CRITICAL (rm-1.x) | 최대 **C** — 성적과 무관하게 실행 전 미리보기+명시 확인. 측정 미달이면 HOLD |
| 2 | heldout_accuracy 미측정(null) ∨ GOLD 앵커 < 하한 | **HOLD** — 내부 추론만, 교사 선택과 비교(Shadow), 자동 제안 없음 |
| 3 | 측정치 ≥ assist 하한 ∧ 비가역 아님(STANDARD) | **A** — 관련 기능 바로 열기·초안·검색·이동 (교사 확인 후 계속) |
| 4 | 측정치 ≥ suggest 하한 | **B** — 후보 2~4 제시 → "이 작업 하시려는 건가요?" → 선택을 교정 evidence로 저장 |
| 5 | 그 외 | **HOLD** |

- ELEVATED(rm)는 egress 경계에서 이미 게이트된 위험 — A/B 산정은 동일하되 실행 정책(§5)에서
  미리보기를 기본 제공한다. 이중 확인 게이트는 두지 않는다(CWAR 희석 금지, rm-1.0 원칙).
- 등급 변화는 승격(§6-6)과 함께만 움직인다 — candidate가 Arena를 통과해 promote될 때
  등급표가 재산출된다. run 이력이 아니라 **최신 promote 시점 스냅샷**에서 파생(replay 안정).
- 실험실 미리보기: 등급표는 대시보드 읽기 전용 카드로 노출 가능(구현은 게이트 후 — HOLD).

## 3. 킨더버스 → 브레인 문맥 계약 (Core/확장 분리 — 절대 규칙 5)

infer 요청의 **Core는 불변**이다(utterance·workspace_context·recent_actions·teacher_context·
session). 전략 문서가 추가로 요구한 문맥은 **runtime_extension**에 배치한다:

| 문맥 | 배치 | 비고 |
|---|---|---|
| 현재 화면·선택 객체·직전 행동 | Core (기존) | workspace_context·recent_actions |
| 직전 대화 | Core (기존) | session.conversation_history |
| **사용자 권한** | runtime_extension (신규) | 실행 정책 판정용 — 추론 점수에 미개입 |
| **현재 반·기관** | runtime_extension (신규) | PII 최소화 — id 참조만, Shadow 0a 거버넌스 §2 적용 |

응답 계약은 ir-0.2로 확장 완료(2026-07-17): `risk_level`(rm 파생 — 후보별 risk_tier와 일치
강제), `required_slots`(**예약** — 슬롯 추출은 트랙 2 구현 전까지 항상 null), `context_used`
(실소비 문맥 에코 — 빈 블록 미포함).

## 4. 연결 준비 일정 (트랙 1과 병행)

1. **계약 동결**: infer 요청/응답(ir-0.2) + runtime_extension 필드 확정 — 본 문서 §3.
2. **Shadow 0a 거버넌스 7항목**(runtime spec §2: 목적·법적 근거·PII 게이트·최소 수집·보존·
   옵트아웃·스키마 매핑) 통과 → 0a 가동(추론 없는 실발화 분포 수집 — Atlas·OOD 검증 공급).
3. **크로스워크 확정**: 킨더버스 표면 ↔ 70의도 매핑(`kinderverse-intent-sync-strategy-v0.1.md`).
4. **통합 리허설**: 스테이징에서 mode=shadow 왕복(계약·지연·버전 기록 재현성 확인 — 서빙 아님).
5. **오픈 게이트 판정**: internal-alignment §2 체크리스트 9항목 + README 게이트(KTIB 80% +
   안전 지표) → 통과 시 의도별 등급표 발행과 함께 단계 개시.

## 5. 실행 정책 (등급 × 액션)

| 액션 유형 | A | B | C(CRITICAL) | HOLD |
|---|---|---|---|---|
| 추천·검색·화면 이동 | 즉시 | 후보 제시 후 | 후보 제시 후 | 없음(내부만) |
| 초안 생성(되돌릴 수 있음) | 즉시 | 확인 후 | 확인 후 | 없음 |
| 실행(발송·기록·삭제) | 확인 후 | 확인 후 | **미리보기+명시 확인 필수** | 없음 |

모든 실행 계열은 교사 확인을 거친다 — A등급의 "즉시"는 비실행 액션(열기·초안·검색)에만
해당한다. 교사의 선택·수정은 전부 교정 evidence로 수집되되, **브레인 변경은 검수·Arena
통과 승격으로만**(수집은 실시간, 변경은 통제 — 트랙 2의 불변 원칙).
