# 시험 문항 만드는 법 (검수자·전문가용)

> 뇌의 실력을 재는 **시험지(KTIB)** 를 만드는 방법이에요.
> 시험이 무엇인지 먼저 알고 싶으면 『서비스 설명서』를, 어느 의도부터 채울지는
> 『CRITICAL 7 출제 가이드』(exam-authoring-critical7.md)를 봐주세요.

---

## 1. 시험 문항이 뭐예요?

문항 하나는 딱 **두 가지**로 이뤄져요:

| | 뜻 | 예 |
|---|---|---|
| **문제** (`teacher_prompt`) | 선생님이 실제로 할 법한 짧고 애매한 말 | 🗣 "지우 어머니한테 오늘 낮잠 못 잤다고 전해줘" |
| **정답** (`intent`) | 63개 의도 중 하나 | 개별 학부모 메시지 (`comm_message_individual_parent`) |

채점: 뇌가 **문제만 보고** 정답을 맞히면 득점. 정답은 뇌에게 숨기고 채점 때만 비교해요.

---

## 2. 만드는 방법은 두 가지

| 방법 | 이럴 때 | 흐름 |
|---|---|---|
| **웹에서 바로** | 자리에서 몇 문항씩 | 화면 상단 **[📄 시험지 검수]** → 문항 작성 탭에서 질문+1~5점 제출 → 다른 사람이 점수 안 보고(blind) 1~5점 → 일치도 통과 시 [등록] |
| **엑셀로 한꺼번에** | 여럿이 나눠 대량으로 | 아래 **§3 따라 하기 ①~④** |

(운영자·스크립트용 YAML 경로는 §8 고급.)

---

## 3. 엑셀 방법 — 따라 하기 ①~④

### ① 양식 받기

| 양식 | 내용 | 언제 |
|---|---|---|
| ⭐ **CRITICAL 7개 먼저** (`ktib_critical7_template.csv`) | 위험 의도 7개 × 30칸 + **'혼동 주의'** 힌트 칸 | **처음이라면 이것부터** (권장) |
| 전체 양식 (`ktib_exam_template.csv`) | 63개 의도 × 10칸 | 7개를 끝내고 넓힐 때 |

- **왜 CRITICAL 7개부터?** 되돌릴 수 없는 위험 의도(부모 전송·출결·삭제)라서 안전
  게이트가 이 7개에 **각 30문항**을 요구해요. 여기가 첫 병목입니다.
- **'혼동 주의' 칸**엔 그 의도와 헷갈리는 이웃 업무가 적혀 있어요 — 그것과 **구별되는**
  질문이 좋은 문항이에요. (의도별 구별 요령: exam-authoring-critical7.md)
- 💻 엑셀은 더블클릭으로 열려요 · 구글 시트는 파일 → 가져오기 → 업로드.

### ② 질문 쓰기 — ✍️ 한 칸만

의도마다 줄이 미리 그려져 있어요. **'시험 질문' 칸에만** 그 의도로 알아들어야 할 말을
**평소 말투로** 쓰세요 (의도 id·뜻 칸은 🔒 그대로). 다 채울 필요 없어요 — 되는 만큼만.

| | 예 | 왜 |
|---|---|---|
| 🟢 좋아요 | "다온이 할머니가 데려가셔서 조퇴야" (출결 기록) | 평소 말투 — '출결'이란 단어 없이도 뜻이 분명 |
| 🟢 좋아요 | "이건 서준이네만 따로 알려야 할 것 같은데" (개별 학부모 메시지) | 혼동 이웃(학급 알림장)과 확실히 구별 |
| 🟢 좋아요 | "아까 그 사진 어디 갔지?" (자료 검색) | 짧고 애매한 진짜 현장 말 — 이런 게 실력을 제대로 재요 |
| 🔴 아쉬워요 | "출결을 기록해줘" | 의도 이름을 그대로 읽은 문장 — 현장에서 이렇게 말 안 해요 |
| 🔴 아쉬워요 | 강화하기·즉석 문답에서 봤던 문장 그대로 | 공부용과 겹치면 자동 거부 (암기 측정 방지) |
| 🔴 아쉬워요 | AI에게 시켜서 만든 문장 | 시험 정답이 AI면 점수가 무의미 — 자동 탐지는 못 하니 **꼭 지켜주세요** |

### ③ 두 분이 O/X 체크

"이 질문이 이 의도의 시험 문제로 적절한가?"를 **서로 상의 없이 각자** O/X로 판정하세요.

- **둘 다 O**인 질문만 등록 대상이 돼요
- 일치도(kappa)는 **자동 계산** — 직접 셈할 필요 없어요
- 전부 똑같이 매기면 일치도를 계산할 수 없어 등록이 안 돼요 — 따로 검수하면 자연히
  몇 개는 갈리니 걱정 마세요

### ④ 올리기 — 검증 → 등록 확정

도움말 → **시험 문항 만들기** 탭의 업로드 폼에서: 검수자 두 분 이름 입력 → 파일 올리기 →
자동 집계(둘 다 O 몇 개·kappa) 확인 → **검증(dry-run)** 통과하면 → **[등록 확정]**.
확정 전엔 아무것도 저장되지 않아요.

## 4. 꼭 지켜야 하는 규칙 (어기면 자동 거부)

1. **`intent`는 63개 의도 id 중 하나** — 없는 것/UNKNOWN 불가 ("정답이 모름"일 수 없음)
2. **발화는 사람이 쓴 진짜 표현** — AI가 지어낸 문장 금지 (넣으면 점수가 무의미)
3. **검수자는 서로 다른 2명** — 같은 사람 두 번은 시스템이 막아요 (별칭·공백 변형도 잡음)
4. **공부용 데이터에 이미 있는 발화 금지** — 겹치면 "암기 측정"이 됨 (자동 거부)
5. **합성 채널 금지** — 컴퓨터가 만든 연습문제는 시험지에 못 들어감 (DB가 물리적으로 차단)

## 5. 몇 문항이면 되나요?

| 목표 | 필요한 문항 | 효과 |
|---|---|---|
| ⭐ **CRITICAL 7개 (첫 목표)** | 의도당 **30문항** | 안전 게이트가 판정을 시작 |
| 한 노드 측정 | 그 의도에 **1문항 이상** | 그 점이 "측정됨" → 어둠에서 벗어남 |
| 점수 안정 | 의도당 **5~10문항** | 1문항이면 0%/100%로 튐 |
| 전 노드 측정 | 63개 전부 (**권장 ~500문항**) | 뇌 전체가 시험 대상 |
| 혼동선 확정 | 같은 혼동 **3번 이상** | 점선 → 실선 깜빡임 |

> 문항은 **재는 도구**예요 — 문항이 늘면 **측정 범위**가 넓어져요. 점수(%)는 공부
> (강화하기·즉석 문답)를 시킨 뒤 **재시험**에서만 올라요. 최종 목표는 전체 80%.

---

## 6. 점수가 오르는 실제 경로

```
문항 확보 → 채점 → 노드가 밝아짐(기본 점수)
        → 선생님들이 강화하기·즉석 문답으로 공부시킴
        → 주 1회 재채점 → 실력 오르면 더 밝게 → … → 전체 80% 🎉
```

## 7. 실행 순서 (운영자)

```
1) 진척 확인    python scripts/ktib_coverage.py     # 의도별 등록 수 · 30까지 남은 갭
2) 시험지 확정  python scripts/run_ktib_build.py --commit
3) 채점 실행    python scripts/run_arena.py --commit   ← 여기서 노드가 밝아짐
```

문항을 추가할 때마다 2~3을 다시 돌리면 시험지가 커지고 더 많은 노드가 측정돼요.
(웹·엑셀 업로드는 등록까지 해주므로 별도 적재 명령이 필요 없어요.)

## 8. 고급 — YAML 파일로 직접 (운영자·스크립트)

시작용 양식: `seeds/ktib_seed_template.yaml`

```yaml
authored_by: "김유아 (○○대 유아교육과 교수, 15년)"   # 필수
approved_by: "박승인 (운영자)"                       # 필수
origin_channel: EXPERT_AUTHORED      # 사람 저작만
dataset_split: BENCHMARK_HOLDOUT     # 시험지 칸

episodes:
  - episode_id: EP_KTIB_0001
    teacher_prompt: "작년 봄 소풍 사진 어디 있지?"   # 문제
    intent: op_workspace_search                    # 정답 (63개 중 하나)
    lang: ko
    reviewers: ["김유아", "이검수"]   # 서로 다른 2명 이상
    agreement_kappa: 1.0             # 두 검수자 일치도
```

적재: `python scripts/ingest_expert_episodes.py --file seeds/ktib_seed_template.yaml --commit`
