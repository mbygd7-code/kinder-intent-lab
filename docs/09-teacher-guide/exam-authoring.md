# 시험 문항 만드는 법 (전문가용)

> 뇌의 실력을 재는 **시험지(KTIB)** 를 만드는 방법이에요.
> 시험이 무엇인지 먼저 알고 싶으면 『서비스 설명서』를 봐주세요.

---

## 1. 시험 문항이란?

문항 하나는 딱 **두 가지**로 이뤄져요:

| | 뜻 | 예 |
|---|---|---|
| **문제** (`teacher_prompt`) | 교사가 실제로 할 법한 짧고 애매한 말 | "작년 봄 소풍 사진 어디 있지?" |
| **정답** (`intent`) | 63개 의도 중 하나 | `op_workspace_search` (자료 검색) |

채점: 뇌가 **문제만 보고** 정답을 맞히면 득점. 정답은 뇌에게 숨기고 채점 때만 비교해요.

---

## 2. 형식 — YAML 파일 한 장

시작용 양식(63개 의도 예시 문항 포함)이 준비돼 있어요: **`ktib_seed_template.yaml`**
(도움말에서 내려받거나 저장소 `seeds/ktib_seed_template.yaml`)

```yaml
authored_by: "김유아 (○○대 유아교육과 교수, 15년)"   # 필수
approved_by: "박승인 (운영자)"                       # 필수
origin_channel: EXPERT_AUTHORED      # 사람 저작만
dataset_split: BENCHMARK_HOLDOUT     # 시험지 칸

episodes:
  - episode_id: EP_KTIB_0001
    teacher_prompt: "작년 봄 소풍 사진 어디 있지?"   # 문제
    intent: op_workspace_search                    # 정답 (63개 중 하나)
    lang: ko
    reviewers: ["김유아", "이검수"]   # 서로 다른 2명 이상
    agreement_kappa: 1.0             # 두 검수자 일치도
```

---

## 3. 꼭 지켜야 하는 규칙 (어기면 자동 거부)

1. **`intent`는 63개 의도 id 중 하나** — 없는 것/UNKNOWN 불가 ("정답이 모름"일 수 없음)
2. **발화는 사람이 쓴 진짜 표현** — LLM이 지어낸 문장 금지 (넣으면 점수가 무의미)
3. **검수자 2명 이상 + 일치도(kappa) 기록** — 시험 정답은 혼자 확정 불가
4. **공부용 데이터에 이미 있는 발화 금지** — 같은 문장을 공부·시험에 겹치면 "암기 측정"이 됨
5. **합성 채널 금지** — `origin_channel`은 `EXPERT_AUTHORED`/`OFFICIAL_CORPUS`/`COMMUNITY_DERIVED`만

---

## 4. 몇 문항을 만들어야 하나?

문항은 **"재는 도구"** 예요. 문항이 많다고 점수가 오르는 게 아니라, **더 많은 노드를 잴 수 있게** 돼요.

| 목표 | 필요한 문항 | 효과 |
|---|---|---|
| **한 노드가 시험을 보게** | 그 의도에 **1문항 이상** | 그 점이 "측정됨" → 어둠에서 벗어남 |
| **점수가 안정적이게** | 의도당 **5~10문항** 권장 | 1문항이면 0%/100%로 튐 — 여러 표현으로 재야 실력이 드러남 |
| **전 노드 측정** | 63개 의도 전부 문항 (**최소 63, 권장 ~500**) | 뇌 전체가 시험 대상이 됨 |
| **한 영역 "가동"(3단계)** | 그 영역 9개 중 **5개 이상** 측정 + 평균 정확도 70%↑ | 영역이 Region Online으로 |
| **혼동선 "확정"** | 같은 혼동이 **3번 이상** 나오게 | 점선 → 실선 깜빡임으로 |

**권장 시작점**: 의도 1개당 8문항 × 63개 ≈ **약 500문항**이면 전 노드가 안정적 점수를 받고 영역·전체 판정이 가능해요. (시작 양식은 의도당 1문항 = 63문항이니, 전문가가 여기서 표현을 늘려주세요.)

## 5. 언제 · 얼마나 자주 하면 점수가 오르나?

**중요: 문항을 늘린다고 점수(%)가 오르지 않아요.** 점수는 뇌가 실제로 맞혀야 올라가요.

```
문항을 늘리면      → 더 많은 노드가 "측정됨"으로 (어둠 → 밝음, 그 노드 실력만큼)
점수를 올리려면    → 공부(강화하기·데이터)를 더 시켜서 뇌 실력을 키운 뒤 → 재시험
```

- **시험 주기**: 기본 **주 1회**(정기). 문항을 추가하거나 공부를 더 시킨 뒤 다시 채점하면 갱신돼요.
- **점수가 오르는 실제 경로**:
  1. 선생님들이 **강화하기**로 약한 노드를 가르침 (공부 ↑)
  2. 주간 시험(Arena)에서 재채점
  3. 실력이 올랐으면 그 노드가 더 밝아지고, 영역·전체 점수가 상승
- **최종 목표**: 전체 첫-의도 정확도 **80%** 도달 (그러면 성장 5단계 "뇌 전체 공명")

즉 **문항(시험지) 확보 → 노드가 밝아지기 시작 → 공부+재시험 반복 → 점수 상승 → 80% 목표**의 순서예요.

## 6. 실행 순서 (운영자)

```
1) 양식 받기      seeds/ktib_seed_template.yaml (또는 도움말에서 다운로드)
2) 문항 채우기    전문가가 발화·정답 검토·교체·추가 (검수자 2인 + kappa)
3) 등록          python scripts/ingest_expert_episodes.py --file seeds/ktib_seed_template.yaml --commit
4) 시험지 확정    python scripts/run_ktib_build.py --commit
5) 채점 실행      python scripts/run_arena.py --commit    ← 여기서 노드가 밝아짐
```

문항을 추가할 때마다 3~5를 다시 돌리면 시험지가 커지고 더 많은 노드가 측정돼요.

## 7. 좋은 문항의 조건

- **짧고 애매하게** — "이거 정리 좀", "아까 그거 어디 갔지?" 같은 진짜 말투. 명확한 명령문보다 애매한 발화가 실력을 제대로 재요.
- **의도마다 여러 표현** — 같은 "자료 검색"도 "찾아줘", "어디 있지?", "그거 못 찾겠어" 등 다양하게.
- **헷갈리는 쌍을 노려서** — 자주 헷갈리는 의도(예: 자료 검색 ↔ 자료 정리)에 문항을 여러 개 넣으면 혼동 관계가 정확히 측정돼요.
- **현장의 진짜 말** — 지어낸 티 나는 문장 대신, 실제로 하거나 들은 표현으로.
